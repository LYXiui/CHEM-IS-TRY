CHEM-IS-TRY 專題簡報演講稿建議總時長：10～12 分鐘（簡報 7～8 分＋現場演示 3～5 分）  啟動演示：`npm run dev` → http://localhost:5173  線上版：https://lyxiui.github.io/CHEM-IS-TRY/---第 1 頁｜封面各位老師、同學好，我是 ________。今天要報告的專題是 CHEM-IS-TRY——  一個用網頁程式設計技術打造的虛擬化學實驗台。---第 2 頁｜專題簡介CHEM-IS-TRY 是一個單頁應用程式（SPA），  整合我們在課堂所學的 HTML、CSS、JavaScript、React 與建置工具。使用者可以透過元素週期表選取元素、從試劑架傾倒溶液，  在實驗台上混合或燃燒，觀看火焰、沉澱、變色、冒泡等動畫，  成功的實驗會自動寫入右側的實驗筆記本，並可一鍵重現。（建議此頁後切換至網頁，做 30 秒介面導覽。）---第 3 頁｜技術一覽專題涵蓋多項網頁技術：  HTML5 負責語意結構；CSS3 呈現動畫與版面；  JavaScript ES6+ 處理模組與邏輯；React 19 以元件化方式組合介面；  Vite 負責開發與打包；Tailwind CSS 輔助響應式排版；  localStorage 則讓筆記與解鎖紀錄保存在瀏覽器本機。---第 4 頁｜HTML5 語意結構網頁從 `index.html` 進入，React 掛載在 `#root` 節點。  我們設定 `lang="zh-Hant"` 指定繁體中文，  並以 `type="module"` 啟用 ES Module，符合現代 JavaScript 模組化寫法。  外部字型則透過 Google Fonts 載入，讓筆記本呈現手寫風格。---第 5 頁｜JavaScript ES6+ 模組化專案將資料與邏輯分離：  元素、化合物、反應表各自放在 `data/` 目錄，  以 `import` / `export` 匯入使用。程式中大量使用箭頭函式、展開運算更新陣列、  解構讀取反應結果，以及可選鏈安全存取筆記快照，  讓程式碼更簡潔、可維護。---第 6 頁｜React 元件化設計介面拆成多個元件：`PeriodicTable` 週期表、`LabBench` 實驗台、  `CompoundAtlas` 合成圖鑑、`Notebook` 筆記本、`ContextMenu` 右鍵說明等。每個 `.jsx` 檔對應一個 UI 模組，  父元件 `App` 持有狀態，以 props 往下傳遞，形成清楚的元件樹與單向資料流。---第 7 頁｜useState實驗台狀態——燒杯內物質、酒精燈、火柴、筆記內容——  都由 `useState` 管理。狀態一改變，React 就自動重新渲染畫面。這是宣告式 UI 的核心：我們描述「現在實驗台長什麼樣」，  而不是手動去改 DOM 每一個節點。---第 8 頁｜useEffect`useEffect` 負責副作用：  網頁一載入，就從 localStorage 還原筆記與已解鎖化合物；  元件卸載時，則清除計時器，避免記憶體洩漏。這讓瀏覽器 API 與 React 生命週期順利結合。---第 9 頁｜useCallback / useRef`useCallback` 快取常用函式，減少不必要的重建；  `useRef` 則保存「不需要觸發重渲染」的資料，  例如實驗過程紀錄 `processLogRef`、碼表計時器 ID。實驗進行中的每一步 log，都累積在 ref 裡，  最後才一次寫入筆記本。---第 10 頁｜Props 與事件處理子元件透過 props 回呼 通知父層：  週期表點選元素 → `onSelect` → 加入燒杯；  按「混合」→ `onMix` → 執行反應。右鍵選單則封裝成自訂 Hook `useContextMenu`，  示範邏輯與 UI 分離的進階寫法。---第 11 頁｜Vite 6 建置工具開發時執行 `npm run dev`，Vite 提供本機伺服器與熱更新（HMR）；  `npm run build` 打包成靜態檔案輸出到 `dist/`；  `npm run preview` 可預覽正式版效果。`vite.config.js` 整合 React 與 Tailwind 外掛，  並設定 GitHub Pages 的部署路徑。---第 12 頁｜Tailwind CSS 4Tailwind 採 Utility-first 寫法，  用 `grid`、`flex`、`gap` 等 class 快速排版。我們同時保留自訂 CSS，  例如筆記本紙張紋理、燒杯反應動畫、火焰與沉澱效果，  讓工具類樣式與專題視覺並存。---第 13 頁｜資料驅動 UI（反應引擎）化學反應以查表法實作：  反應物排序後組成鍵值，例如 `'H+O'` 對應生成水。`getReaction()` 查表回傳產物與現象（沉澱、變色、氣體等），  UI 依結果渲染動畫與報告。  新增反應只需擴充資料，不必改動元件結構——  這就是 Model 與 View 分離 的實踐。系統涵蓋手動設計反應與 118 種元素自動生成反應，  並以 `npm run test:all` 驗證化學式與反應覆蓋。---第 14 頁｜localStorage筆記本與解鎖紀錄以 JSON 字串存入 localStorage，  重新開啟網頁仍能保留學習成果。筆記超過十筆時，會改為下拉選單列出實驗化學式，  點選後可查看過程並重現實驗。---第 15 頁｜自訂 Hook 與模組拆分`useLabState.js` 集中管理實驗狀態與反應流程；  `reactionAnim.js` 把現象對應到燒杯動畫；  `compoundHints.js` 驅動合成圖鑑提示。這種關注點分離讓各模組職責清楚，方便擴充新反應與新動畫。---第 16 頁｜專案目錄結構專案根目錄有 `index.html` 與 `vite.config.js`；  `src/` 底下分 `components`、`hooks`、`data`、`utils`；  `presentations/` 存放本簡報的 Markdown 原始檔。執行 `npm run ppt` 可從 Markdown 自動產生 PPTX。---第 17 頁｜建置與簡報安裝依賴後，`npm run dev` 開發、`npm run build` 建置正式版。  簡報本身也用 Marp CLI 從 Markdown 匯出，  示範「文件即簡報」的現代工作流程。---第 18 頁｜線上體驗 QR Code專題已部署至 GitHub Pages。  請掃描畫面上的 QR Code，或開啟網址  https://lyxiui.github.io/CHEM-IS-TRY/即可在手機或電腦上體驗：週期表選元素、試劑混合、  燒杯內變色與沉澱動畫，以及實驗筆記重現功能。（可現場掃碼開啟，做 1 分鐘快速演示。）---第 19 頁｜謝謝聆聽CHEM-IS-TRY 將化學學習與網頁程式設計結合，  以動畫與筆記強化學習回饋。  本程式僅供教學模擬，真實實驗仍須遵守實驗室安全規範。以上是我的報告，謝謝聆聽！---附錄｜建議現場演示（3～5 分鐘）順序	實驗	操作摘要1	Mg + O 燃燒	週期表選 Mg、O → 火柴 → 燃燒2	Zn + HCl 產氫	燒杯 → HCl(aq) → Zn →
